# План тестирования мобильного приложения "Мобильный хоспис"

## 1. Обзор приложения
- Назначение: работа с новостями хосписа
- Основные модули: новости, цитаты, информация о приложении
- Целевая платформа: Android API 29+

## 2. Границы тестирования
### Включается:
- UI тестирование основных экранов
- Функциональность авторизации
- Работа с новостями (просмотр, создание, редактирование)
- Просмотр цитат
- Навигация между разделами

### Исключается:
- Тестирование на других версиях Android
- Performance тестирование
- Security тестирование

## 3. Типы тестирования
- Функциональное тестирование
- UI тестирование
- Smoke тестирование
- Регрессионное тестирование

## 4. Стратегия автоматизации
- Используемые инструменты: Espresso, JUnit, Allure
- Подход: Page Object Pattern
- Окружение: эмулятор Android API 29

## 5. Этапы выполнения
1. Анализ требований и планирование
2. Составление чек-листа и тест-кейсов
3. Настройка проекта для автотестов
4. Написание UI-тестов
5. Подключение Allure для отчетов
6. Сравнение ручного и автотестирования
